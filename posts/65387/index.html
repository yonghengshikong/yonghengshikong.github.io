ECMAScript新特性总览

1. ECMAScript 1 (ES1) - 1997

  • 首个版本：定义了基本的 JavaScript 语法和功能，奠定了 JavaScript 标准的基础。

2. ECMAScript 2 (ES2) - 1998

  • 小幅更新：对 ES1 进行了一些格式化的调整，以匹配 ISO/IEC 16262 国际标准，基本没有新功能。

3. ECMAScript 3 (ES3) - 1999

  • 扩展语言特性：添加了许多新功能，包括正则表达式支持、更好的错误处理机制、try/catch 异常处理、Array 和 String 方法（如 map、filter 和 split）。
  • 标志性的语言特性：如 Object、Array 等的标准化。

4. ECMAScript 4 (ES4) - 未发布

  • 开发中止：ES4 原计划加入大量新特性，例如模块、类、接口和类型检查。但由于复杂度和各方意见不一致，最终未发布。

5. ECMAScript 5 (ES5) - 2009

  • 关键性更新

    • 严格模式：通过 "use strict" 引入严格模式，以增强代码的安全性和性能。
    • 原生 JSON 支持：加入 JSON.parse 和 JSON.stringify。
    • 增强对象和数组方法：如 Object.create、Object.keys、Array.isArray、forEach、map、filter、reduce。
    • 属性描述符：允许通过 Object.defineProperty 来控制对象属性的可写性、可枚举性等。
    • 增强数组的处理能力：扩展了数组的高阶函数。

6. ECMAScript 6 (ES6/ES2015) - 2015

  • 重大更新

    ：引入了现代 JavaScript 的大部分关键特性。

    • **let 和 const**：块级作用域变量声明。
    • 箭头函数：简化函数表达式的写法，同时绑定 this。
    • 模板字符串：通过反引号 ``` 来进行多行字符串和嵌入变量。
    • 解构赋值：支持对象和数组的解构赋值。
    • 默认参数：为函数参数设置默认值。
    • 展开运算符（Spread Operator）：简化数组和对象的合并。
    • Promise：处理异步操作的新方法。
    • 模块化：引入 import 和 export，标准化模块管理。
    • 类（Class）：支持基于原型的类语法。
    • Symbol：新的原始数据类型，用于创建唯一的标识符。

7. ECMAScript 2016 (ES7) - 2016

  • 少量新特性

    • 指数运算符：使用 ** 来代替 Math.pow。

  • Array.prototype.includes：检查数组是否包含某个元素，比 indexOf 更加直观。

8. ECMAScript 2017 (ES8) - 2017

  • 异步函数：支持 async/await，使异步代码更接近同步写法。
  • Object.entries 和 Object.values：提取对象的键值对和所有值。
  • 字符串填充方法：padStart 和 padEnd。
  • 共享内存和原子操作：增加 SharedArrayBuffer 和 Atomics 对象，允许更高效的并行编程。

9. ECMAScript 2018 (ES9) - 2018

  • Rest/Spread 运算符扩展：支持在对象中的展开和剩余参数语法。
  • 异步迭代器：引入 for await...of 循环。
  • Promise.finally：在 Promise 完成后执行指定代码。
  • 正则表达式增强：添加了命名捕获组、后行断言（lookbehind）、dotAll 模式等。

10. ECMAScript 2019 (ES10) - 2019

  • Array.prototype.flat 和 flatMap：将多维数组展平。
  • Object.fromEntries：将键值对列表转换为对象。
  • 字符串修剪方法：trimStart 和 trimEnd。
  • 可选捕获绑定：try/catch 中可以省略 catch 的参数。
  • Function.prototype.toString 修复：输出函数代码的完整实现。

11. ECMAScript 2020 (ES11) - 2020

  • 可选链（Optional Chaining）：使用 ?. 避免访问空值对象属性时报错。
  • 空值合并运算符（Nullish Coalescing）：?? 在 null 或 undefined 时返回默认值。
  • **动态 import()**：允许动态加载模块，返回 Promise。
  • BigInt 类型：处理超过 Number 的安全范围的大整数。
  • Promise.allSettled：等待所有 Promise 完成，无论成功还是失败。

12. ECMAScript 2021 (ES12) - 2021

  • 逻辑赋值运算符：&&=, ||=, ??= 等赋值简写形式。
  • **String.prototype.replaceAll**：允许替换所有匹配项。
  • WeakRefs 和 FinalizationRegistry：对垃圾回收的支持，允许在对象被清理前执行回调。
  • Promise.any：只要有一个 Promise 成功则返回。

13. ECMAScript 2022 (ES13) - 2022

  • at() 方法：Array 和 String 的 .at() 方法支持负索引访问。
  • **Object.hasOwn()**：比 Object.prototype.hasOwnProperty 更简洁的对象属性判断。
  • 类字段与私有属性：类内直接定义字段，私有属性以 # 开头。
  • 正则表达式匹配索引：matchAll 中捕获匹配的起始和结束索引。

14. ECMAScript 2023 (ES14) - 2023

  • **Array.prototype.toSorted()**：不修改原数组的情况下排序。
  • **Array.prototype.toReversed() 和 toSpliced()**：不修改原数组，返回翻转或删除后的副本。
  • Symbol.symbols：一种新的内置 Symbol，允许标记不可修改的字段。
