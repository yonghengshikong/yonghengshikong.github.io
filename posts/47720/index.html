TypeScript 中的 接口 (Interface) 详解

接口 (Interface) 是 TypeScript 中用于定义对象形状的一种方式。它能够定义对象的结构、类的行为以及函数的签名等，是 TypeScript 类型系统的一个重要组成部分。接口提供了一种契约或规范，来描述对象和类必须遵守的结构或行为。

在 TypeScript 中，接口的主要作用是声明对象和类的结构，使得代码更具可读性和可维护性。

1. 基本的接口定义

接口的最基本的形式就是通过 interface 关键字定义一个对象的类型。你可以指定对象的属性和它们的类型。

2. 可选属性

在接口中，我们可以使用 ? 来标记某个属性为可选的。这意味着这个属性可以在对象中不存在。

在这个例子中，address 是一个可选属性，所以可以选择提供，也可以不提供。

当你不知道对象的确切属性名时，可以使用索引签名来描述对象的结构。索引签名允许你指定对象的属性类型。

6. 类实现接口

接口不仅可以描述对象的结构，还可以作为类的类型约束。在 TypeScript 中，类可以通过 implements 关键字来实现接口，从而强制类符合接口的结构。

在这个例子中，Dog 类实现了 Animal 接口，确保类中有 name 属性和 sound 方法。使用 implements 关键字，我们强制类遵循接口的约定。

7. 继承接口

接口可以继承其他接口。这是通过 extends 关键字来实现的，允许一个接口继承多个其他接口的属性和方法。

8. 接口与联合类型

接口还可以与联合类型结合使用，以便定义多种不同类型的组合。你可以使用接口来描述联合类型中的某个部分。

9. 接口的扩展与多重继承

接口不仅可以继承一个接口，也可以继承多个接口，支持多重继承，允许你将多个接口的功能组合到一个接口中。

在这个例子中，C 接口继承了 A 和 B，因此它需要包含 a、b 和 c 这三个属性。

10. 接口与类型别名的区别

接口与类型别名在 TypeScript 中有许多相似之处，但它们也有一些区别：

  • 扩展性：接口是可以被继承的，并且可以通过声明合并扩展接口。类型别名则不能被直接继承。
  • 功能限制：类型别名不仅可以定义对象的结构，也可以定义其他类型（如联合类型、交叉类型、元组等），而接口主要用于描述对象和类的形状。

这两者看起来相似，但接口通常用于类和对象的定义，而类型别名则更灵活，能支持更复杂的类型结构。

总结

  1. 接口 (Interface) 是 TypeScript 中的强大功能，用于定义对象、类、函数等的结构。
  2. 接口可以包含属性、方法、可选属性、只读属性等。
  3. 类实现接口，可以确保类的结构符合接口的要求。
  4. 接口继承可以将多个接口的功能合并。
  5. TypeScript 中还有 **类型别名 (type alias)**，它与接口有许多相似之处，但类型别名可以定义更多复杂的类型结构。

通过合理使用接口，可以增强 TypeScript 代码的可读性、可维护性和类型安全性，从而提升开发效率和代码质量。
