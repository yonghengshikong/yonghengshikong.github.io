深入理解JavaScript原型链-类-函数与对象的关系

深入解析：JavaScript 原型链、类、对象与继承

JavaScript 中的原型链、类、对象与继承是理解语言核心机制的关键，它们构成了 JS 面向对象编程（OOP）的基础。本文将全面解析这些概念，包括 函数、对象、构造函数、原型链、__proto__、prototype、继承 等，并通过代码实例详细讲解其机制。

1. 函数的本质

在 JavaScript 中，函数既是可执行的代码块，也是对象。每个函数都有一个独特的 prototype 属性，这是面向对象的关键点。

  • greet 是一个函数，同时也是一个对象。
  • greet.prototype 是一个对象，用于定义共享给所有实例的方法和属性。
  • 每当函数被用作构造函数（通过 new 操作符），新实例的原型链将连接到这个 prototype 对象。

4. __proto__ 与 prototype 的关系

4.1 概念区别

  • **__proto__**：每个通过 new 操作符创建的对象都拥有一个隐藏属性 __proto__，它指向创建该对象的构造函数的 prototype 属性。

  • **prototype**：

    prototype 是每个函数对象（包括类）自动拥有的一个属性（仅函数和类持有），它指向一个对象，这个对象用于存储可以由该函数实例共享的属性和方法。

    作用：

    • 定义共享方法或属性： 所有由该函数（作为构造函数）创建的实例都可以通过原型链访问 prototype 对象上的属性和方法。
    • 建立原型链： 实例通过其内部的 [[Prototype]]（即 __proto__）属性链接到其构造函数的 prototype。

4.2 验证关系

类的 prototype 的 constructor 属性默认指向类本身。这是 JavaScript 的一个设计特点，用于确保类和它的原型之间的逻辑关系一致。

constructor 是 prototype 对象上的一个属性，用于指向创建该 prototype 对象的构造函数。

对于类或函数来说，它的 prototype.constructor 默认指向自身，这确保了我们可以通过实例的原型链找到对应的构造函数。

4.3 原型链

通过 __proto__，对象可以沿着原型链查找属性和方法.

实例对象的__proto__指向的是类或函数的prototype,然后子类的prototype.__proto__指向父类的prototype,最后指向Object.prototype,然后Object.prototype取Object.prototype.__proto__结束。因为Object.prototype.__proto__为null

5.2 使用 prototype 实现继承

在 ES6 之前，使用 prototype 手动实现继承：

Dog.prototype = Object.create(Animal.prototype); 和 Dog.prototype.__proto__ = Animal.prototype; 的效果在设置原型链时是类似的，但它们的实现机制和使用场景有所不同。下面详细对比和解释：

1. Dog.prototype = Object.create(Animal.prototype);

行为

  • Object.create(Animal.prototype) 创建了一个新的对象，这个对象的原型（即 __proto__）是 Animal.prototype。
  • 然后，将这个新对象赋值给 Dog.prototype。

结果

  • Dog.prototype 成为一个独立的对象，它的原型是 Animal.prototype。
  • 改动 Dog.prototype 不会影响 Animal.prototype，因为它们是两个不同的对象。

示例

2. Dog.prototype.__proto__ = Animal.prototype;

行为

  • 直接设置 Dog.prototype 的 __proto__ 指向 Animal.prototype。
  • 改动的是已有的 Dog.prototype 对象的内部原型链，而不是创建一个新对象。

结果

  • Dog.prototype 本身没有被替换，而是修改了它的内部链接（__proto__）。
  • 如果之前对 Dog.prototype 做过自定义操作，这些修改会保留。

示例

差异与推荐使用

操作 Object.create __proto__
是否创建新对象 是（Object.create 会创建一个新对象） 否，修改的是现有对象的 __proto__

为什么常用 Object.create？

  • 清晰：Object.create(Animal.prototype) 明确表示你希望创建一个基于 Animal.prototype 的新对象。
  • 符合规范：避免直接操作 __proto__，因为 __proto__ 是非标准的历史遗留属性，ES6 引入了更优雅的方式（如 Object.create 和 Object.setPrototypeOf）。
  • 安全性：确保 Dog.prototype 独立于 Animal.prototype，减少意外副作用。

5.4 Object.setPrototypeOf 的意义

Object.setPrototypeOf 是 JavaScript 提供的一种方法，用于动态设置一个对象的原型（[[Prototype]]）。它改变目标对象的内部原型指向，从而影响该对象的继承关系。

语法

  • **obj**：要修改其原型的目标对象。
  • **prototype**：目标对象的新原型，可以是对象或 null（如果为 null，表示该对象不继承任何对象）。

返回值：返回修改后的目标对象。

作用

  1. 动态改变对象的原型链：
    • 改变一个对象的继承关系，让它从另一个对象继承属性和方法。
  2. 修复或调整继承链：
    • 在运行时更改对象的原型，比如修正错误的原型设置。

示例

  • 优点
  1. 动态灵活：
    • 可以在运行时修改对象的继承关系，而不需要在声明时确定。
  2. 兼容性较好：
    • ES6 正式标准化了 Object.setPrototypeOf，是推荐的方法之一（相较于直接操作 __proto__）。
  • 缺点
  1. 性能问题：
    • 改变原型链可能导致性能下降，因为 JavaScript 引擎需要重新计算和优化对象的属性查找路径。
    • 如果频繁使用，可能会显著降低代码性能。
  2. 潜在风险：
    • 动态修改原型链可能引入难以调试的错误，破坏了代码的可预测性和稳定性。

推荐使用场景

  1. 修复继承关系：
    • 在继承关系设置错误时（如 prototype 错误覆盖），用来修复或重设原型链。
  2. 动态调整继承：
    • 在某些场景需要动态扩展或切换继承关系时。

避免使用的场景

  1. 频繁调用：
    • 不要在性能敏感的代码中频繁调用 Object.setPrototypeOf，因为它会显著降低 JavaScript 引擎的优化效果。
  2. 可以通过构造函数或 Object.create 实现的场景：
    • 如果可以通过声明时确定继承关系，优先使用 Object.create 或直接设置 prototype。

Object.setPrototypeOf 是一种功能强大的工具，用于动态设置或修复对象的原型链。它比直接操作 __proto__ 更符合规范，也更安全。但由于性能和可维护性问题，应尽量在明确的需求场景下使用，而不是滥用。

8. 总结

  1. Object.create() 的意义：
    • 创建一个继承自指定原型的对象。
    • 不会执行构造函数，是轻量级的继承方法。
  2. new 的意义：
    • 调用构造函数创建对象。
    • 同时执行构造函数初始化逻辑。
  3. Object.create() 和 new 的区别：
    • Object.create() 适合需要手动设置原型链、不需要初始化的场景。
    • new 适合结合初始化和原型链继承的场景。
  4. 避免错误：
    • Animal.prototype 通常是普通对象，不能用 new 调用。
    • 使用 Object.create() 可以更灵活地设置原型关系，而无需调用构造函数。
