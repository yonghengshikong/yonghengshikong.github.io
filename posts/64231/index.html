ECMAScript1-5新特性详解

ECMAScript 1 到 ECMAScript 5 新特性详解

ECMAScript 1 (ES1) - 1997

主要特性

1. 基本语法和控制结构

  • ECMAScript 1 引入了变量声明、条件判断、循环等基础控制结构，JavaScript 程序从此可以实现基本的逻辑控制。

2. 数据类型

ES1 定义了基本的数据类型，包括 Number、String、Boolean 和 Object。

3. 全局函数

ES1 提供了 parseInt 和 parseFloat 函数，用于将字符串解析为整数和浮点数。

ECMAScript 2 (ES2) - 1998

ES2 是对 ES1 的规范化和修订版本，没有新特性。主要目的是与 ISO 标准保持一致。

ECMAScript 3 (ES3) - 1999

主要特性

1. 正则表达式（RegExp）

  • 解释：正则表达式用于字符串模式匹配和替换。
  • 用法：/pattern/flags 的形式或通过 RegExp 构造函数创建。

2. 异常处理 (try-catch-finally)

  • 解释：try-catch-finally 结构允许捕获错误并进行相应处理，finally 总是会执行。
  • 用法：在 try 块中编写可能引发异常的代码，catch 捕获异常信息，finally 执行必要的清理工作。

3. 数组方法

  • 解释：ES3 引入了 push、pop、shift 和 unshift 等数组操作方法。

4. for...in 循环

  • 解释：遍历对象的所有可枚举属性。
  • 用法：for...in 遍历对象键值对。

5. Function.apply 和 Function.call

  • 解释：apply 和 call 允许将不同的 this 值传递给函数，并立即调用函数。apply 使用数组传递参数，而 call 逐个传递参数。

ECMAScript 5 (ES5) - 2009

主要特性

1. 严格模式（Strict Mode）

  • 解释：严格模式通过 "use strict"; 指令启用，更严格地解析代码，禁止一些不安全的语法。
  • 用法：在文件或函数顶部添加 "use strict";，启用严格模式。

2. JSON 支持

  • 解释：JSON.parse 和 JSON.stringify 用于处理 JSON 数据格式。
  • 用法：JSON.parse 将 JSON 字符串转换为对象；JSON.stringify 将对象转换为 JSON 字符串。

3. 对象增强方法

  • 解释：Object.create、Object.keys 和 Object.defineProperty 等方法增强了对象操作的灵活性。

4.2. Array.prototype.forEach()

forEach(callback(currentValue [, index [, array]])[, thisArg]) 方法对数组中的每个元素执行一次提供的函数。

  • 参数:

    • callback

4.3. Array.prototype.map()

map(callback(currentValue [, index [, array]])[, thisArg]) 方法返回一个新数组，其结果是通过对原数组的每个元素调用提供的函数得到的。

  • 用于对数组中的每个元素进行转换或操作。

  • 返回一个新数组，该数组的每个元素都是对原数组相应元素经过某种处理后的结果。

  • 参数:

    • callback

4.4. Array.prototype.filter()

filter(callback(element [, index [, array]])[, thisArg]) 方法创建一个新数组，包含所有通过测试的元素。

  • 用于从数组中筛选出满足特定条件的元素。

  • 返回一个新数组，该数组只包含满足条件的元素。

  • 参数:

    • callback

4.11. Array.prototype.splice()

splice(start, deleteCount[, item1[, item2[, ...]]]) 方法可以添加、删除或替换数组中的元素。它会修改原数组，并返回被删除的元素（如果有）。

  • 参数:
    • start: 从该索引开始修改数组。
    • deleteCount: 要删除的元素数量。
    • item1, item2, ...（可选）: 要添加到数组的新元素。(需要注意的是插入的位置是在删除的位置)
  • 返回值:
    • 返回包含已删除元素的数组，如果没有删除元素，则返回空数组。

示例:

4.16. Array.prototype.sort()

sort(compareFunction) 方法是用于对数组中的元素进行排序的一个内置方法。它会改变原数组并返回引用相同的数组。默认情况下，sort() 方法将元素按字符串Unicode编码进行排序。要进行数值排序或自定义排序，可以提供一个比较函数。

  • 参数:
    • compareFunction（可选）: 一个用于比较两个元素的函数。如果提供该函数，数组会根据其返回值进行排序。
      • 如果返回值小于 0，则 a 在 b 之前。
      • 如果返回值大于 0，则 a 在 b 之后。
      • 如果返回值等于 0，则 a 和 b 的相对位置保持不变。
  • 返回值:
    • 返回已排序的数组（是原数组的引用）。
1.默认行为

当没有提供 compareFunction 时，sort() 方法将元素转换为字符串，然后按字符顺序进行排序。例如，数字会被转为字符串并按字典序排列。

4.排序的稳定性

在 ECMAScript 2019（ES10）之前，sort() 方法的排序算法是不稳定的。这意味着如果两个元素相等，它们的相对顺序可能会被改变。从 ECMAScript 2019 开始，排序的稳定性得到了保证，但具体实现仍然依赖于 JavaScript 引擎。

5.注意事项
  1. 原地排序: sort() 方法会改变原数组，而不是创建一个新数组。
  2. 返回值: sort() 方法返回对原数组的引用。
  3. 空数组: 对于空数组，sort() 方法不会有任何效果。
6.小结
  • Array.prototype.sort() 是一个强大而灵活的方法，适用于多种排序需求。
  • 使用自定义比较函数可以轻松实现数字、字符串和对象的排序。
  • 理解默认排序行为非常重要，以避免在处理数字时产生意外结果。
