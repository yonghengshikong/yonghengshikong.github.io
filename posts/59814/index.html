十、NodeJs mongoDB®详解

一、简介

MongoDB 是一个面向文档的 NoSQL 数据库，以高性能、高可用性和自动扩展著称。它使用 BSON（一种类似 JSON 的二进制格式）存储数据，使得数据结构更加灵活。MongoDB 不使用关系型数据库的表结构，而是使用集合（Collection）和文档（Document），通过键值对存储数据，适用于大数据和实时数据处理等需求。

官方地址 https://www.mongodb.com/

MongoDB 设计的特点：

  • 文档存储：数据以文档（Document）形式存储，灵活且易于扩展。
  • Schema-less：允许文档结构随需求调整，方便处理非结构化和半结构化数据。
  • 分布式架构：支持自动分片（Sharding）和高可用性，适合扩展到大量节点。
  • 强大查询语言：支持聚合、筛选、排序、投影等多样查询方式。

二、核心概念

  • 数据库 (Database)
    MongoDB 的数据库相当于关系型数据库中的数据库，里面包含多个集合（表）。每个 MongoDB 实例可以创建多个数据库，每个数据库有独立的权限控制。

  • 集合 (Collection)
    集合类似于关系数据库中的表，但不强制要求文档的 Schema 相同，因此可以在同一集合中存储不同结构的文档。集合在数据库中，使用时需指定数据库。

  • 文档 (Document)
    文档是 MongoDB 中的基本数据单元，相当于关系数据库中的一行记录。每个文档是一个 BSON 格式的对象，结构类似于 JSON。例如：

    这里的 _id 字段是每个文档的唯一标识，如果不指定，MongoDB 会自动生成。

  • 字段 (Field)
    字段是文档中的键值对，类似于关系数据库中的列。一个文档可以包含多个字段，并且字段类型可以不同，例如字符串、数值、数组、嵌套文档等。

  • 索引 (Index)
    MongoDB 支持为集合创建索引，常见的是单字段索引、复合索引和文本索引等。索引提高查询速度，但会占用额外空间。

  • 副本集 (Replica Set)
    MongoDB 中的一组节点，负责数据的冗余备份。副本集中包含一个主节点和多个从节点，主节点负责写操作，从节点同步数据，确保高可用性。

  • 分片 (Sharding)
    MongoDB 的水平分片策略，将数据分散到不同的分片中，使系统具备良好的扩展性。

6. 条件运算符

Mongoose 支持丰富的查询条件运算符，用于复杂的数据查询。常见的条件运算符如下：

条件运算符 说明 示例
$eq 等于某个值 { age: { $eq: 18 } }
$ne 不等于某个值 { age: { $ne: 18 } }
$gt 大于某个值 { age: { $gt: 18 } }
$gte 大于等于某个值 { age: { $gte: 18 } }
$lt 小于某个值 { age: { $lt: 18 } }

7. 索引和聚合

1.索引

Mongoose 支持为字段创建索引以加速查询。常见索引类型有唯一索引、复合索引和文本索引。

优点：

查询性能：在复杂的查询中使用其他字段（例如用户名、电子邮件等）作为筛选条件，MongoDB 如果没有针对这些字段建立索引，就必须扫描整个集合，导致性能下降。为频繁查询的字段创建索引可以显著提升效率。

复合查询：例如，在查询中需要组合多个字段，如按 name 和 age 共同筛选用户，这种情况下可以创建复合索引 { name: 1, age: 1 } 来加速联合查询。

唯一约束：业务需求可能要求某些字段具有唯一性（如电子邮件或用户名），可通过 unique 索引来确保。

排序需求：如果某字段常用于排序，创建索引能优化排序操作。默认 _id 索引只能对 _id 字段排序，如果我们想按 createdAt 排序，就需要为 createdAt 字段创建索引。

全文搜索：MongoDB 提供了文本索引来支持复杂的全文检索，适用于字段内容较长或频繁进行模糊查询的情况。

索引类型 说明 示例
单字段索引 为单个字段创建索引 { name: 1 }
复合索引 为多个字段组合创建索引 { name: 1, age: -1 }
唯一索引 保证字段值的唯一性 { email: { unique: true } }
文本索引 支持全文搜索 { description: 'text' }
稀疏索引 忽略 null 和缺失的字段 { phone: { sparse: true } }

2.聚合

Mongoose 提供的聚合框架可以对数据进行复杂的统计分析，常用的聚合管道操作符包括 $match、$group、$sort、$limit 等。

操作符 说明 示例
$match 筛选符合条件的文档 { age: { $gt: 18 } }
$group 分组统计 { _id: "$hobbies", total: { $sum: 1 } }
$sort 排序 { age: -1 }
$limit 限制结果数量 5
$project 控制输出字段 { name: 1, age: 1 }

示例：按爱好分组并统计每个爱好的人数

以下是一个详细的表格，列出了 Mongoose 中常见的验证类型及其参数说明和示例：

验证类型 参数 描述 示例
必填 required 确保字段必须存在 { required: true }
最小长度 minlength 限制字符串的最小长度 { minlength: 5 }
最大长度 maxlength 限制字符串的最大长度 { maxlength: 100 }

数值最小值 min 限制数值的最小值 { min: 18 }
数值最大值 max 限制数值的最大值 { max: 65 }
自定义验证 validate 使用自定义函数验证 { validate: customValidatorFn }
唯一值 unique 确保字段在集合中唯一 { unique: true }
默认值 default 设置字段的默认值 { default: '未设置' }
枚举值 enum 限制字段的值为预定义的一组值 { enum: ['user', 'admin', 'guest'] }
数值类型 type 指定字段的数据类型 { type: Number }
日期类型 type 指定字段为日期类型 { type: Date }
数组类型 type 指定字段为数组类型 { type: [String] }
对象类型 type 指定字段为对象类型 { type: Object }

这个模式使用了各种验证类型来确保用户数据的有效性，展示了如何在 Mongoose 中使用这些验证规则。

validate参数说明

  1. validator:
    • 类型: Function
    • 描述: 自定义验证函数，接受一个参数 v（要验证的值）。返回 true 表示验证通过，返回 false 表示验证失败。
  2. message:
    • 类型: String | Function
    • 描述: 当验证失败时，返回的错误信息。可以使用字符串，也可以使用一个函数，函数接受一个参数 props，其中包含被验证的值（props.value）、字段路径（props.path）、模型实例等信息。

自定义验证的注意事项

  • 如果在模式中为字段设置了 required: true，则该字段的值必须存在，验证会在保存文档之前进行。
  • 自定义验证在保存时会触发，如果自定义验证失败，Mongoose 将抛出一个错误，包含所有验证失败的字段信息。
  • 使用自定义验证时，确保验证逻辑高效，以免在大量数据操作时影响性能。
