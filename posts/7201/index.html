TypeScript中的类详细解析

TypeScript 中的类、约束、继承、抽象类和接口：详细解析

在 TypeScript 中，类、继承、约束、抽象类 和 接口 是构建面向对象程序的重要概念。这些概念提供了更强的类型安全性、更灵活的结构，并能够帮助开发者在编写代码时避免很多常见的错误。本篇博客将详细介绍这些概念，并解答一些常见的疑问，帮助你理解它们如何共同工作。

1. 类（Class）

类是面向对象编程的核心，它定义了对象的结构和行为。TypeScript 中的类在 ES6 类的基础上进行了增强，支持静态成员、访问修饰符（如 public, private 和 protected）以及构造函数等。

类的基本语法

  • 构造函数：用于初始化对象实例。
  • 成员变量：类中定义的属性。
  • 方法：定义类的行为。

2. 继承（Inheritance）

继承是面向对象编程的核心特性之一，它允许我们从一个已有的类创建一个新类，并继承父类的属性和方法。通过继承，可以重用代码并且增强其功能。

继承的基本语法

  • **super()**：子类必须调用 super() 来调用父类的构造函数。
  • 方法重写：子类可以重写父类的方法。

分析

1.C++ 中的行为分析

C++ 的继承机制和成员变量覆盖

在 C++ 中，继承关系是静态的。子类会覆盖父类的成员变量，如果子类定义了一个与父类同名的成员变量，那么子类的成员变量会遮蔽（隐藏）父类的成员变量。换句话说，父类和子类各自有独立的成员变量，即使它们同名。

C++ 中的成员函数调用与对象绑定

在 C++ 中，当你调用一个父类的成员函数时，实际的行为依赖于 静态绑定 和 动态绑定 的机制。

  1. 静态绑定：如果你调用的是父类的成员函数，而该函数没有使用 virtual 关键字（非虚函数），那么函数调用就是静态绑定。函数调用是在编译时就确定下来的，不能通过子类覆盖。
  2. 动态绑定：如果父类的成员函数是 virtual（虚函数），那么调用的成员函数是动态绑定的，具体调用哪个版本的函数是通过 虚函数表（VTable） 实现的，在运行时通过对象的实际类型来确定。

  • 分析：

    1. 父类构造函数：在 C++ 中，子类的构造函数调用之前，父类的构造函数会先被调用。在 Parent 类的构造函数中，name 被设置为 "Parent"，但此时 Child 类中的 name 成员已经被初始化为 "child"，因此父类构造函数修改的是父类的 name，而不是子类的 name。
    2. 成员覆盖：在 C++ 中，父类和子类都有一个名为 name 的成员变量。由于子类的 name 会覆盖父类的 name（即 隐藏 父类的成员变量），在访问 childInstance.name 时，访问的是子类的 name，即 "child"。
    3. **调用 updateName**：当你调用 childInstance.updateName("Updated by Parent") 时，实际上调用的是父类的 updateName 方法，因为该方法在 Parent 类中定义，而子类没有重写它。由于该方法修改的是父类的 name，它会更新父类的 name 为 "Updated by Parent"，但是此时 修改的是父类的 name 成员，而非子类的 name。因此，输出的是 "child"。

    总结：

    在 C++ 中，即使子类覆盖了父类的成员变量，父类方法调用时修改的也是父类的 name，而非子类的 name。这是因为成员变量的覆盖并不影响父类的成员。除非调用的是虚函数，他才会根据虚函数表去修改对应实例对象的值。

2.JavaScript 中的继承与方法调用

JavaScript 的继承机制和 C++ 有显著不同。在 JavaScript 中，所有对象的属性和方法都是通过原型链来查找的，而且 JavaScript 的类并不像 C++ 那样直接将成员变量分配到类的实例中。成员变量实际上是类实例的属性，并且原型链的行为使得调用父类的方法时，子类可以覆盖和修改父类的属性。

在 JavaScript 中，当子类实例化时，父类的构造函数会在子类的构造函数中调用（如果子类没有显式调用 super()，会自动调用）。不过，父类构造函数的调用时机是：在子类构造函数的成员变量初始化之前，但通常会在子类的 this 上下文可用之前调用。

更详细的解释：

  1. 如果子类没有定义构造函数： 当子类没有定义构造函数时，JavaScript 会自动调用父类的构造函数。并且，父类的构造函数会在子类的成员变量初始化之前被调用。

  2. 如果子类显式定义了构造函数： 当子类显式定义了构造函数时，必须在子类构造函数内调用 super() 来调用父类的构造函数。super() 必须在子类构造函数中调用，在子类访问 this 之前。否则，JavaScript 会抛出错误。

在上面的例子中，你可以看到：

  • super() 调用是在 Child 类的构造函数中，且它发生在对 this 的访问之前。这样，父类的构造函数会首先被调用，子类再对自己的成员变量进行初始化。

  • 如果子类没有显式定义构造函数，会自动调用父类的构造函数，且父类的构造函数会在子类的成员变量初始化之前调用。

  • 如果子类显式定义了构造函数，必须显式调用 super()，并且 super() 必须在访问 this 之前调用。

总结：C++ 和 JavaScript 的差异

  1. **C++**：
    • 在 C++ 中，父类和子类的成员变量是完全独立的，子类可以覆盖父类的成员，但方法操作的是父类的成员变量（即使是通过子类实例调用父类方法）。
    • C++ 中的继承机制更多依赖于成员的直接访问和静态绑定。
  2. JavaScript：
    • 在 JavaScript 中，父类和子类的成员变量（或者对象属性）是共享的，子类的成员变量可以覆盖父类的成员变量。
    • 方法的调用基于原型链，动态绑定使得子类可以覆盖父类的方法，且修改的成员变量是子类实例的属性。
    • 需要注意的是，无论父类在构造函数中如何修改成员变量，都会被子类重新定义的成员变量给覆盖，无论子类是否有显性的调用构造函数。
    • 子类重写成员变量时，不能只声明不定义，要声明和定义一起实现。

因此，C++ 和 JavaScript 在继承和成员变量的覆盖上有显著差异，导致它们在行为上的不同。

3. 抽象类（Abstract Class）

抽象类是无法直接实例化的类，它只能作为其他类的基类。抽象类通常包含 抽象方法（没有实现的方法），这些方法必须在继承该抽象类的子类中实现。抽象类提供了一种通用的方式来定义一组类的基本结构。

抽象类的基本语法

  • 抽象方法：在抽象类中定义，但没有提供实现。子类必须提供实现。
  • 普通方法：可以在抽象类中定义，并提供实现。子类可以继承或重写。

abstract 关键字

abstract 关键字用于标记类或方法为抽象的。抽象类不能直接实例化，必须通过继承来创建实例。

4. 接口（Interface）

接口是 TypeScript 的一种用于定义对象结构的方式，它定义了对象的 类型，但并不提供实现。接口可以包含方法签名、属性声明和索引签名。接口与抽象类的一个主要区别是，接口只定义结构，不允许有实现。

接口的基本语法

  • 实现接口：通过 implements 关键字，类可以实现一个或多个接口。
  • 接口的可选属性：接口中的属性可以是可选的，通过 ? 标记。

抽象类与接口的区别

  • 接口：接口只能定义方法签名和属性，没有方法实现。接口不关心类的继承结构。
  • 抽象类：抽象类可以包含实现的方法，也可以定义抽象方法。抽象类是具有部分实现的类。

6. 总结

  • 类（Class） 是面向对象编程的基本单位，用于定义对象的属性和行为。
  • 继承（Inheritance） 允许一个类继承另一个类的属性和方法，并可进行方法重写。
  • 抽象类（Abstract Class） 是不能直接实例化的类，主要用于定义其他类的基础结构。
  • 接口（Interface） 主要用于定义对象的结构，类可以实现接口并提供方法实现。
  • 类型约束（Constraints） 用于限制泛型类型，确保其具有特定的属性或方法。
  • as 关键字 用于进行类型断言，将值强制转换为指定的类型。
